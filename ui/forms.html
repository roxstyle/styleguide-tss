Forms

Default Styles using a grid

All forms are given default styles to present them in a readable and scalable way. Styles are provided for text inputs, select lists, textareas, radio buttons and checkboxes, and buttons.

The default layout for labels is above their inputs, unless you have a simple form with a couple of fields like a login. For questions regarding default form UI, please read Luke Wroblewskis' Web Form Design.

This sample is using the grid for form columns, based on Foundation Zurb Grid

Default Styles in a flow

The same default styles using an unordered list and wrappers

The default layout for labels again, is above their inputs, unless you have a simple form with a couple of fields like a login. There is an alternative for inline labels

Notes:

  1. This sample is using wrapper classes in an unordered list. It uses less markup and classes.
  2. Part of the consideration of what style to use for a forms depends on the form complexity and size. Some forms use an outer grid with nested flow wrappers. You will need to test your forms to decide what works best for the fields.
  3. In both the flow and grid layouts - neither configuration uses a label tag wrapped around the input.
  4. Remember that the label for="some-id"... is looking for input id="some-id".../
  5. We are still using xml endings in markup, at this point.

Inline Labels

For specific cases where you want inline labels

The first sample is using grid classes. Using a grid size for the single label and single input. Put the label inside a different column to the left of the input. Then add a class of .right to the label to have it align to the right. Adding a class of .inline will have it vertically center against an input.

This works responsively provided the label is short, and it is a single instance.

The second example uses flow wrappers. This can be used for multiple dropdown filters above dynamic tables. The wrappers will keep the label and dropdown connected as the layout changes responsively. Again, the preferred layout is label above input. The inliine label is for specific case only.

Notes:

  1. The inline label does not work as well for associating the label with its input.
  2. The inline label does not work as well for larger forms in columns where the labels may be varying lengths.
  3. The inline label will not work responsively as well to keep the label and input associated.

Actions can be tied to buttons, and prefix / postfix labels. Use div class="row collapse" to create label / action / input combinations. Use columns to define the size of the pre/postfix span class="postfix" or span class="prefix".
